# FlightScanner v1.4.0 功能更新

**发布日期**: 2026-04-07

**版本**: v1.4.0

---

## 版本概述

v1.4.0 引入 **AI 进化引擎（Evolution Engine）**，实现 4 齿轮闭环自进化预测系统：AI 不再只是生成一次性简报，而是通过记录预测、回测结果、诊断失误、反馈改进四个环节持续学习，预测质量随使用时间增长。

同时完成历史监控页交互重设计，将原来的双页跳转改为 `st.tabs` 内联标签，消除页面导航状态复杂度。

---

## 主要功能：AI 进化引擎

### 架构概览

```
每次采集        每天 UTC 03:00          生成 AI 简报时
    │                  │                      │
[G1 执行器] → [G2 监控器] → [G3 诊断器] → [G4 进化器]
记录预测快照   回测 Pain Index  LLM 根因分析   注入历史上下文
                                            + 信誉 badge
```

### G1 执行器

每次采集完成后，满足以下任一条件时自动记录一次结构化预测快照到 `ai_prediction_log` 表：

- 已累积 ≥ 5 个不同采集批次（数据里程碑）
- 最新批次价格与历史中位数偏差 ≥ 8%（价格异动）

冷却规则：同一路线 **12 小时内只记录一次**。

记录字段：预测时间、基准价格、建议操作（Buy/Wait）、趋势、置信度、建议原因、LLM 来源、距出发天数。

### G2 监控器

每天 UTC 03:00 定时运行，对以下记录执行回测：

- **航班已出发**：该路线所有 `pending` 记录均可评估（完整出发前价格窗口）
- **预测满 7 天**：仅该路线**最新一条** `pending` 记录触发（防止历史旧记录批量触发）

Pain Index 计算：

| 场景 | 计算方式 |
|------|----------|
| 建议 Buy，后续价格下跌 > 5% | `(基准价 - 实际最低价) × 0.7`（机会成本折扣） |
| 建议 Wait，后续价格上涨 > 5% | `(实际最终价 - 基准价) × 1.0`（全额错过成本） |
| 价格变动 ≤ 5% | `neutral`，pain = 0 |

结果状态：`win` / `loss` / `neutral` / `skipped`（数据不足）

附加字段 `catchable_low_exists`：检测观测窗口内是否存在可捕捉低价（需连续 ≥ 2 批次低于基准 5%）。

### G3 诊断器

对 `pain_index > 200 CNY` 且尚未分析的 `loss` 记录发起第二次 LLM 调用，输出结构化根因分析：

- `error_category`：7 分类枚举（`Holiday_Surge_Ignored`、`Airline_Flash_Sale_Missed`、`Too_Close_To_Departure`、`Trend_Reversal_Undetected`、`Low_Historical_Data`、`External_Event_Impact`、`Other`）
- `rca_analysis`：100 字以内的中文分析与改进建议

### G4 进化器

每次生成 AI 简报时实时注入历史失误摘要到 system prompt，并在 UI 上展示信誉徽章：

**历史上下文注入**（已评估记录 ≥ 3 条时生效）：
```
【历史预测记录】胜率 60%（10 次）
近期失误：
  - 建议Buy，实际痛苦指数 ¥240，分类：Holiday_Surge_Ignored
请在本次分析中参考以上历史失误，避免重复类似错误。
```

**信誉徽章颜色**：胜率 ≥ 70% 绿色 / 50%~70% 黄色 / < 50% 红色

> 胜率 = `(win + neutral × 0.5) / evaluated_count`（neutral 算半胜）

**熔断机制**：近期连续 3 次 `pain_index ≥ 300 CNY` 时触发熔断，UI 暂停自动生成并显示警告，用户可点击「强制生成」覆盖。

## 历史监控页重设计

### 变更内容

将 v1.3.0 引入的双页跳转方案替换为 `st.tabs` 内联标签：

**旧方案（已移除）**：
- `st.session_state["_page"]` 路由状态
- `_render_history_entry()` 底部入口卡片
- `_render_history_page()` 独立历史页
- 「查看历史 →」/ 「← 返回主页」导航按钮

**新方案**：
- 路线区域直接渲染两个标签：`当前监控` / `历史记录（N）`
- 历史路线数量作为标签 badge 实时更新
- 无页面跳转，无 session state 路由管理
- 完全消除返回按钮失效问题

## 数据库变更

新增 `ai_prediction_log` 表（`CREATE TABLE IF NOT EXISTS`，旧数据库启动时自动创建）：

| 字段组 | 字段 |
|--------|------|
| G1 | `predicted_at`, `price_at_prediction`, `days_until_flight`, `recommended_action`, `reason`, `trend`, `confidence`, `llm_source` |
| G2 | `outcome_status`, `actual_min_price`, `actual_final_price`, `pain_index`, `catchable_low_exists` |
| G3 | `rca_run_at`, `error_category`, `rca_analysis` |
| 索引 | `ix_aiplog_route_predicted (route_id, predicted_at)`, `ix_aiplog_outcome (outcome_status)` |

---

## 升级说明

**新数据库**：`init_db()` 自动建表，无需手动操作。

**已有数据库**：`_apply_migrations()` 在启动时幂等追加 `CREATE TABLE IF NOT EXISTS`，已有表静默跳过。

```bash
# 重启即可
PYTHONPATH=/path/to/FlightScanner venv/bin/streamlit run ui/app.py
```
