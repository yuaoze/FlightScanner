# FlightScanner v1.5.1 补丁更新

**发布日期**: 2026-04-30

**版本**: v1.5.1

---

## 版本概述

修复 AI 价格简报的刷新策略：每次 Streamlit 重启都会重新生成一次简报，导致前端加载慢且白白消耗 AI token。改为**数据指纹**作为缓存键 + **磁盘持久化**，只有新采集数据到达时才判断是否重新生成。

---

## 修复内容

### 问题背景

v1.2.0 为 AI 简报加了"12 小时时间窗"缓存（`cache_key = f"ai_brief_{route.id}_{_12h_window_id()}"`），试图每 12 小时才重算一次。但两个问题：

1. **时间窗独立于数据**：即便采集间隔是 6 小时，12 小时到了不管有没有新数据都会触发重生；反之采集到异动数据后还要等到下一个窗口边界才刷新
2. **`st.session_state` 和 `@st.cache_data` 都是内存级**：Streamlit 服务重启（开发时频繁）后全部清空，首次渲染必定命中 `cache_key not in st.session_state` 分支 → 跑 `_should_auto_trigger` → 调 DeepSeek API → **每次 app 启动烧一次 token**

### 修复：数据指纹 + 磁盘缓存

**数据指纹** ([`ai_brief.py`](../ui/components/ai_brief.py))

```python
def _data_fingerprint(price_history):
    latest = max(price_history, key=lambda fp: fp.scraped_at)
    return f"{latest.batch_id}|{latest.scraped_at}|{len(price_history)}"
```

以"最新采集批次 ID + 时间戳 + 总记录数"作为指纹。数据不变 → 指纹不变 → 缓存命中；一旦新批次到达，指纹自动变化，触发条件判断。

**磁盘持久化**

```
.cache/ai_briefs/route_{route_id}.json
  └── {"fingerprint": "...", "brief": {...}}
```

每次生成/更新简报时异步写盘，三个小 helper：`_load_brief_from_disk`、`_save_brief_to_disk`、`_clear_brief_from_disk`，全部 try/except 兜底（磁盘异常不影响渲染）。

**渲染流程**

```
render_ai_brief(route, price_history):
  current_fp = _data_fingerprint(price_history)

  # 1) session_state 未命中时从磁盘 hydrate（跨重启的关键步骤）
  if cache_key not in st.session_state:
      disk = _load_brief_from_disk(route.id)
      if disk:
          st.session_state[cache_key] = disk["brief"]
          st.session_state[fp_key]    = disk["fingerprint"]

  # 2) 数据指纹未变 → 快路径直接渲染，无触发判断、无 spinner、零 token
  if cache_key in st.session_state and st.session_state[fp_key] == current_fp:
      _render_brief_card(...); return

  # 3) 数据变化时评估触发条件
  should_trigger, reason = _should_auto_trigger(price_history)
  if should_trigger:
      brief = _cached_generate_brief(...)          # 调 AI
      st.session_state[cache_key] = brief
      st.session_state[fp_key]    = current_fp
      _save_brief_to_disk(...)
  elif cache_key in st.session_state:
      # 有新数据但不满足触发条件 → 保留旧简报，只更新指纹
      st.session_state[fp_key] = current_fp
      _save_brief_to_disk(route.id, current_fp, st.session_state[cache_key])
  else:
      # 无缓存无触发 → 手动按钮
      ...
```

**"🔄 重新生成"按钮**

同时清内存、磁盘和 `@st.cache_data`：

```python
st.session_state.pop(cache_key, None)
st.session_state.pop(fp_key, None)
_cached_generate_brief.clear()
_clear_brief_from_disk(route.id)
st.rerun()
```

## 效果对比

| 场景 | v1.5.0 前 | v1.5.1 |
|------|---------|--------|
| Streamlit 冷启动 | 首次渲染调 AI + spinner + token 消耗 | 磁盘命中直接渲染（零 token，瞬完成） |
| 12 小时边界到达无新数据 | 触发重算 | 指纹不变，继续复用旧简报 |
| 采集到新批次，但不满足触发条件 | 等 12h 窗口 | 立即更新指纹，保留旧简报 |
| 采集到新批次且满足触发条件 | 等下次渲染 | 立即重算（spinner 仅此场景出现） |
