# FlightScanner v1.4.1 补丁更新

**发布日期**: 2026-04-08

**版本**: v1.4.1

---

## 版本概述

三项 bug 修复：

1. **历史路线不再被自动采集**：`target_date < today` 的路线从调度器和手动采集中完全排除
2. **去哪儿 Cookie 失效不再卡住前端**：移除阻塞式交互提示，改为非阻塞警告 flash
3. **G1 执行器 AI 调用修复**：`asyncio.run()` 嵌套事件循环问题，改为直接 `await` 协程

---

## 修复 1：历史路线停止采集

### 问题

`target_date` 已过期的路线（航班已出发）仍会被 APScheduler 调度和执行采集，浪费资源，也不会产生有意义的数据。

### 修复内容

**`src/flightscanner/scheduler/price_monitor.py`**

- `reschedule_all_routes()`：遍历路线时增加 `target_date < date.today()` 跳过判断，历史路线不再注册定时任务
- `scrape_this_route()` 闭包（`schedule_route()` 内部）：触发条件从 `is_active` 扩展为 `is_active AND target_date >= today`，兜底拦截在运行期间到期的路线

**`ui/components/overview.py`**

- 历史路线卡片不再渲染 🔄 立即采集按钮（`target_date < today` 时该列留空）

**`tests/test_registry.py`**

- `test_job_fetches_fresh_route_from_db` mock 补充 `fresh_route.target_date = date.today()`，修复因新增 `target_date` 检查导致的 `TypeError`

## 修复 2：Cookie 失效不再卡住前端

### 问题

去哪儿采集返回 0 条结果时，`_maybe_refresh_and_retry()` 通过 `asyncio.run_in_executor` 调用 `input()` 等待用户输入，在 Streamlit（非 TTY）环境中导致前端界面完全卡死。

### 修复内容

**`src/flightscanner/scrapers/qunar_scraper.py`**

- `_maybe_refresh_and_retry()` 移除全部交互逻辑（`isatty` 检测、`run_in_executor + input()`、Cookie 刷新重试）
- 改为直接记录 `logger.warning` 并立即返回 `[]`，不再阻塞事件循环

**`src/flightscanner/scheduler/price_monitor.py`**

- `__init__` 新增 `self._scrape_warnings: List[str] = []`
- `scrape_route()` 每次开始时清空该列表
- `_scrape_oneway()` 中当某爬虫平台返回空列表时，追加平台级警告（如 `"去哪儿 未获取到数据，Cookie 可能已失效"`）

**`ui/app.py`**

- `trigger_immediate_scrape()`：采集结束后检查 `monitor._scrape_warnings`，若非空则将 flash 级别设为 `warning`，消息包含失效平台名及刷新 Cookie 的操作指引
- flash 渲染处新增 `elif level == "warning": st.warning(msg)` 分支（原来只有 success / error）

## 修复 3：G1 执行器 DeepSeek 调用 asyncio 嵌套问题

### 问题

G1 执行器 `_maybe_log_prediction()` 是 `async def`，运行在 APScheduler 的事件循环中。它调用同步函数 `generate_brief_with_fallback()`，后者内部通过 `asyncio.run()` 启动协程。由于调用时事件循环已在运行，`asyncio.run()` 抛出异常，被外层 `except` 捕获后显示为 "降级到规则引擎"，实际上是 AI 调用从未真正执行。

### 根本原因

```
APScheduler 事件循环（已运行）
  └─ await _maybe_log_prediction()        ← async def
       └─ generate_brief_with_fallback()  ← 同步函数
            └─ asyncio.run(coroutine)     ← 💥 禁止在运行中的事件循环内创建新循环
```

### 修复内容

**`src/flightscanner/analyzers/deepseek_analyzer.py`**

- 新增 `generate_brief_with_fallback_async()`：与同步版本逻辑完全相同，但将 `analyzer.generate_brief_sync()`（内含 `asyncio.run()`）替换为直接 `await analyzer.generate_brief()`，不创建新事件循环

**`src/flightscanner/scheduler/price_monitor.py`**

- `_maybe_log_prediction()` 改为 `await generate_brief_with_fallback_async()`

同步版 `generate_brief_with_fallback()` 保持不变，Streamlit UI 层继续使用原版，两条调用路径互不干扰。
