# FlightScanner v1.3.0 功能更新

**发布日期**: 2026-04-03

**版本**: v1.3.0

---

## 版本概述

v1.3.0 包含两项主要新功能及若干 UI 精修：

1. **精准航班号监控**：新增按航班号精确追踪指定航班价格与状态的监控模式
2. **历史监控页**：主页改为只展示未来路线，过期路线迁入独立历史页，统计卡片仍基于全量计算

---

## 新功能 1：精准航班号监控

### 功能概述

新增 `monitoring_mode = "flight"` 模式。用户可在「🎯 精准监控」弹窗中指定具体航班号（如 CA953），系统将在每次采集时精确匹配该航班的票价与可用状态，不再保存无关航班的数据。

### 使用方式

点击主页顶部「🎯 精准监控」按钮，通过 3 步弹窗完成添加：

- **第 1 步**：填写出发/目的地城市、日期，选择「搜索选择」或「手动输入」模式
- **第 2 步**（搜索模式）：从实时搜索结果列表中选择去程/回程航班；（手动模式）直接输入航班号，支持选择舱位
- **第 3 步**：设置目标价格、采集间隔、采集上限，点击「开始精准监控」

### 航班状态跟踪

每次采集后自动更新航班状态，路线卡片标题与展开内容均展示：

| 状态 | 含义 |
|---|---|
| ✓ 有票 | 航班存在且有余票 |
| ⚠️ 售罄 | 航班找到但 `seatsLeft = 0` |
| ⚠️ 未找到 | 本次搜索结果中未出现目标航班号 |
| ⚠️ 时刻变动 | 实际起飞时间与添加时参考时刻偏差超过 60 分钟 |

往返路线：去程/回程任一为 `sold_out` 或 `not_found` 时路线整体标记为对应状态。

### 技术实现

**数据库** (`src/flightscanner/models/database.py`)

`Route` 表新增 7 列（自动迁移，兼容旧数据库）：

| 列名 | 类型 | 说明 |
|---|---|---|
| `monitoring_mode` | TEXT NOT NULL DEFAULT 'route' | `'route'` \| `'flight'` |
| `outbound_flight_no` | TEXT | 去程航班号，如 `CA953` |
| `inbound_flight_no` | TEXT | 回程航班号（往返时使用） |
| `pinned_seat_class` | TEXT | 指定舱位，`NULL` 表示不限 |
| `outbound_dep_time_ref` | TEXT | 添加时的去程起飞参考时刻 `HH:MM` |
| `inbound_dep_time_ref` | TEXT | 添加时的回程起飞参考时刻 `HH:MM` |
| `last_flight_status` | TEXT | 最近一次采集后的航班状态 |

**采集逻辑** (`src/flightscanner/scheduler/price_monitor.py`)

- `scrape_route()` 检测到 `monitoring_mode == "flight"` 时，调用 `_scrape_pinned_flights()` 替代普通路由采集
- `_scrape_pinned_flights()`：对单程路线做一次单程搜索，对往返路线分两次单程搜索（去程 + 回程），各取最低价匹配
- `_match_pinned_flight()`：精确匹配航班号（不区分大小写），支持舱位过滤，返回 `(best_match, status)`
- `_determine_flight_status()`：综合去程/回程状态判断最终状态，优先级：`sold_out > not_found > schedule_changed > available`
- `_time_diff_minutes()`：辅助函数，计算两个 `HH:MM` 字符串的时间差（分钟）

**服务层** (`src/flightscanner/core/services/route_service.py`)

- `RouteWithLatestPrice` 数据类新增 7 个精准监控字段
- `add_route()` 接受新增参数并写入数据库
- `get_all_routes()` SELECT 子查询包含新增字段
- 新增 `update_flight_status(route_id, status)`：更新单条路线的 `last_flight_status`

**UI** (`ui/app.py`)

- 新增 `_show_add_pinned_flight_dialog()`：3 步弹窗，`step` 通过 `session_state["_pf_step"]` 管理
- 新增 `_search_flights_for_pinned()`：弹窗内同步搜索，`max_results=100`，调用完毕关闭浏览器
- 新增 `_format_flight_option()`：格式化航班选项字符串（航班号 + 航司 + 时刻 + 价格 + 来源）
- 新增 `_clear_pf_dlg_state()`：取消弹窗时清理 `_pf_*` 相关 session state
- 主页顶栏新增「🎯 精准监控」按钮，布局从 `[8,2,1]` 调整为 `[6,2,2,1]`

**测试**

- 新增 `tests/test_pinned_dialog_logic.py`：弹窗状态机逻辑单元测试
- 新增 `tests/test_price_monitor_pinned.py`：`_match_pinned_flight()`、`_determine_flight_status()` 单元测试

## 新功能 2：历史监控页

### 功能概述

主页路线列表改为仅展示 `target_date >= today` 的**未来路线**。当数据库中存在已过期路线时，主页底部自动出现「历史记录」入口卡片；点击「查看历史 →」通过 session state 切换到历史页（无浏览器跳转）。

### 主页变化

- 路线列表：只显示出行日期 ≥ 今天的路线
- 统计卡片（总路线数/活跃监控/达标/即将出行）：仍基于**全量路线**计算
- 无过期路线时：底部入口不渲染，不占空间
- 有过期路线时：底部出现分隔线 + 白色入口卡，显示「N 条出行日期已过期的监控路线」

### 历史页

- 顶部白色 header 条：「🕐 历史监控」标题 + 记录数 chip
- 左侧「← 返回主页」按钮，点击切换回主页
- 路线卡片列表：复用 `render_route_list()`，样式与主页一致，全部操作（暂停/采集/删除/间隔调整）均正常
- 历史页「立即采集」触发信号：Drain 逻辑遍历全量路线，历史页操作也能正常触发

### 技术实现 (`ui/app.py`)

- `st.session_state["_page"]`：页面路由状态，`"main"` 或 `"history"`，首次进入默认 `"main"`
- 主页深色 banner + 操作按钮组：加 `if current_page == "main":` 条件，历史页不渲染
- 路线分割：在 `with get_session()` 内一次性查询全量 `routes`，再按 `target_date` 分为 `upcoming_routes` / `historical_routes`
- Flash message 移入 session block（在 drain 之后、路由分发之前）
- 新增 `_render_history_entry(historical_routes)` / `_render_history_page(historical_routes, svc)`

**新增 CSS** (`_inject_css()`)

- `.fs-history-entry`：主页底部入口卡片（白色圆角卡 + hover 阴影）
- `.fs-history-header` / `.fs-history-header-title` / `.fs-history-header-sub`：历史页顶部 header 条

## UI 精修

### `_combine_roundtrip_prices()` 重构 (`scheduler/price_monitor.py`)

旧逻辑：若发现含 `return_flight_info` 的已组合记录则直接返回，否则进行去程/回程配对。两者互斥，无法同时处理。

新逻辑：分别收集已组合记录（`combined_existing`）和待配对单程记录（`outbound` + `returns`），配对后合并为 `all_combined`，按价格升序返回。Qunar 国际往返降级搜索返回的单程对与携程已组合记录可并存处理。

### Ctrip 往返价格去重 (`scrapers/ctrip_scraper.py`)

修复往返价格解析时对 `adultTax` 的重复叠加：

- `roundTripPrice` / `twowayPrice` / `totalAdultPrice` / `roundPrice` 等字段已是含税总价，标记 `price_is_total = True`，不再叠加 `adultTax`
- 仅当降级为 `adultPrice`（基础票价）时才叠加 `adultTax`

### 路线卡片 UI (`ui/components/overview.py`)

- 平台最新价展示由 `st.metric()` 改为自定义 HTML `.fs-price-row` 行，价格字号更大，涨跌颜色更直观
- metadata 行由 `st.caption()` 改为 `.fs-chip` chip 标签组
- 精准监控路线展开后额外显示：航班号 chip（`.fs-chip-accent`）、起飞参考时刻、舱位、航班状态 badge
- 统计卡片顶部色条从左侧竖条改为顶部横条（3px），移除 emoji 图标
- 新增 `_FLIGHT_STATUS_BADGE` 字典和 `_flight_status_badge()` 辅助函数

### 趋势图平台颜色 (`ui/components/charts.py`)

图例颜色固定：去哪儿 `#3d7ff5`（蓝）、携程 `#12b76a`（绿），不随数据顺序变化。

### AI 简报标题 (`ui/components/ai_brief.py`)

「AI 价格简报」标题由 `st.markdown("#### ...")` 改为 `<p class="fs-section-subtitle">` 以统一样式。

## 升级说明

本版本包含数据库 schema 变更，新增 7 列精准监控字段。

**新数据库**：`init_db()` 自动建表，无需手动操作。

**已有数据库**：`_apply_migrations()` 在启动时幂等执行，自动 `ALTER TABLE` 添加缺失列，旧路线默认 `monitoring_mode='route'`，行为与历史版本完全一致。

```bash
# 重启即可（迁移自动执行）
PYTHONPATH=/path/to/FlightScanner venv/bin/streamlit run ui/app.py
```
