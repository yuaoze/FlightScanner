# FlightScanner v2.0.0 重大升级

**发布日期**: 2026-05-11

**版本**: v2.0.0

---

## 版本概述

v2.0 是一次**前端架构整体重写**：把 Streamlit 的"数据面板"风格升级为 React + TailwindCSS 构建的现代「购票决策助手」。底层爬虫、调度器、分析器、通知器**完全保留**（v1.4.2 / v1.5.x 的所有修复都继承），但渲染层、API 层、状态管理全部翻新。

副带修复了一批之前 Streamlit 时代潜伏的 bug：时区显示错乱、Route 删除卡住、定时调度器漏启动、SQLite 外键不强制导致孤儿数据等。

---

## 一、整体架构

### 旧（v1.x）

```
Streamlit (ui/app.py) ──直连──> SQLAlchemy ──> SQLite
                       └─触发─> PriceMonitorScheduler
```

### 新（v2.0）

```
React + Vite (frontend/)            ←─ TailwindCSS v4 / Framer Motion / Recharts
        │ HTTP /api/*
        ▼
FastAPI (src/flightscanner/api/)    ←─ uvicorn, lifespan 启动 PriceMonitorScheduler
        │
        ▼
SQLAlchemy ──> SQLite (WAL + 外键强制)
```

- **API 层**: `src/flightscanner/api/` —— routers / schemas / 时区工具 / 路线过滤器 / 状态解析器
- **前端**: `frontend/` —— Vite + React 18 + TypeScript + TailwindCSS v4 + Framer Motion + Recharts + @tanstack/react-query + react-router-dom
- **保留 Streamlit**: 仅作为兼容备选，所有新开发只在 React 版本里进行
- **后台调度器随 API 启动**: FastAPI lifespan event 自动 `PriceMonitorScheduler.start()`，不再需要单独启动 Streamlit

## 五、亮点功能详解

### 5.1 周末灵感雷达（v1.5.0 全新视觉重构）

- **沉浸式 Hero**：深紫渐变背景 + 旋转扫描扇形 + 三层同心圆脉冲 + 漂浮 emoji
- **精选评分** [`_featured_score`](src/flightscanner/api/routers/radar.py)：
  - 价格梯度（<¥600 +35 / <¥1000 +25 / ...）
  - 击败均价（≥85% +30 / ≥70% +22）
  - 跨境免签 +18 / 落地签 +10
  - AI 内容质量 +6 / 红眼惩罚 -12
- **多样化** ≤2 张/目的地，避免被某城市数据包刷屏
- **真实城市图**：Unsplash CDN，遮罩+渐变保证文字可读
- **一键锁定为监控**：自动写时间窗（19:00 起飞 / 18:00-23:59 回程）+ 立即触发采集

### 5.2 路线详情 4 Tab

- **价格走势**：30 天小时级 + 多平台分色折线（去哪儿橙 / 携程蓝 / Trip 绿）+ 目标价参考线 + 月历热力图
- **AI 洞察**：预测时间线（Win/Loss/Pending 状态点）+ 路线级胜率
- **航班列表**：批次下拉（含采集时刻）+ Top-N 切换（5/10/20/50）
- **监控设置**：目标价 / 间隔 / 时间窗 8 字段 + "立即采集" + 删除确认

### 5.3 设置中心可编辑

- **25 项配置实时编辑**：写入 `.env` + 热更新 singleton（无需重启即生效）
- **底部固定操作条**：dirty 时浮起，含「放弃」+「保存」双按钮
- **凭据管理**：SMTP / Telegram / WeCom / Feishu / DeepSeek API key 全部 password 输入，留空保留原值
- **校验前置**：服务端用 pydantic 验证器（如 `scraper_type` 必须是 `qunar/ctrip`），错误返 400 + 中文 detail
- **Cookie 卡片**：扫码刷新（base64 二维码 + 轮询）/ 手动粘贴上传（JSON 数组或原始字符串）/ 清除

### 5.4 监控总览倒计时

- `RouteResponse` 新增 `latest_scraped_at` (ISO UTC) + `scrape_interval`
- 前端 [`useTicker`](frontend/src/hooks/useTicker.ts) hook 60s 心跳触发 rerender
- [`nextScrapeCountdown`](frontend/src/lib/utils.ts) 计算 `⏱ 2h 35m 后采集` / `即将采集` / `等待首次采集` / `已暂停`

### 5.5 时间窗即时生效

- ConfigTab 改 `dep_time_from / ret_dep_time_from` 等 8 字段 → PUT 写库
- **历史数据立即按新窗口重新筛选**显示（chart / sparkline / latest_price / volatility / flights 全部应用 [`filter_history_by_route`](src/flightscanner/api/route_filter.py)）
- 后续采集自然走调度器原有的 `_apply_route_filters`，约束一致

### 5.6 数据分析口径统一

- **修复前**: Analytics `total_routes` 只看 `is_active=1`（含已过期未关闭路线）→ 与同页 volatility/trend 列表不一致
- **修复后**: 三处 KPI 全部使用 `is_active AND target_date >= today` 的严格定义
- 抽出 [`status_resolver.py`](src/flightscanner/api/status_resolver.py) 让 routes/stats/analytics 用同一份 AI-aware 状态映射（之前 stats 用简陋规则、routes 用 AI，counts 跟 cards 显示不一致）

## 六、底层修复（与新前端无关，但本次版本一并升级）

| 修复 | 文件 | 说明 |
|------|------|------|
| **时区**：UTC↔CST 显示错乱 | `api/time_utils.py` + 全部 router | 数学字段返 ISO UTC 带 `Z`，展示字段服务端转 CST |
| **调度器自启** | `api/main.py` lifespan | 之前依赖 Streamlit 启动，纯 API 进程时调度器静默不跑 |
| **NOT NULL 删除报错** | `database.py` Route↔NotificationLog cascade | `backref(cascade="all, delete-orphan")`，删 Route 不再 500 |
| **datetime naive vs aware** | `scheduler/price_monitor.py:1155` | `_compute_recent_3d_low` 比较前补 tzinfo |
| **SQLite 外键强制** | `database.py` `connect` event | `PRAGMA foreign_keys=ON`，防止采集协程在路线删除后写孤儿行 |
| **周末雷达扫描清理回归** | `scheduler/price_monitor.py:1812` | 仅在 deals 非空时清旧记录，避免 0 结果覆盖有效缓存 |
| **雷达去重** | `api/routers/radar.py` | 按 `(destination, outbound_date)` 取最新一条；chip 计数用 `count(distinct destination)` |
| **批次累积** | `_run_weekend_radar_batch` | 扫前删该周末旧数据，防止重复扫描堆积 |

## 七、新增数据库对象

### 7.1 `NotificationLog` 表（v2.0 新增）

```python
class NotificationLog(Base):
    __tablename__ = "notification_log"
    id              = Column(Integer, primary_key=True)
    route_id        = Column(Integer, ForeignKey("routes.id"), nullable=False)
    notified_at     = Column(DateTime, default=utcnow)
    price           = Column(Numeric(10, 2))
    trigger_reason  = Column(String(50))   # target_hit/below_avg/...
    channel         = Column(String(50))   # email/telegram/wecom/feishu
    status          = Column(String(20))   # success/failed
    message_summary = Column(Text)
    route = relationship("Route", backref=backref("notification_logs", cascade="all, delete-orphan"))
```

`scheduler/price_monitor.py::_send_alert` 接受 `route_id` + `trigger_reason` 后写入。

## 十、启动方式变化

### 旧 (v1.x)

```bash
streamlit run ui/app.py    # 一条命令
```

### 新 (v2.0)

```bash
# 终端 1：API + 调度器
python -m uvicorn flightscanner.api.main:app --host 127.0.0.1 --port 8000

# 终端 2：前端 dev server
cd frontend && npm install && npm run dev
# 访问 http://localhost:5173
```

生产部署：`npm run build` 后用 nginx serve `frontend/dist/` 并代理 `/api` 到 uvicorn。

旧的 Streamlit (`ui/app.py`) 仍可运行作为兼容备选，但不再增加新功能。
