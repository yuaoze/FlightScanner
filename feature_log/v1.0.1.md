# FlightScanner v1.0.1 优化更新

**发布日期**: 2026-02-28

**版本**: v1.0.1

## 版本概述

FlightScanner v1.0.1 是对 v1.0.0 的重要优化更新，主要解决了两个关键用户体验问题：
1. 增强了监控灵活性 - 支持每条路线单独设置采集间隔，并提供调试模式立即采集
2. 优化了登录流程 - 无头模式下自动弹出二维码并等待登录完成，无需手动重启

## 主要功能优化

### 1. 灵活的监控间隔设置

#### 问题背景
v1.0.0 版本使用全局6小时的固定采集间隔，所有路线使用相同的监控频率，无法满足差异化需求：
- 紧急路线需要更频繁的监控
- 非紧急路线可以降低监控频率以减少系统负载
- 开发调试时需要立即触发采集

#### 解决方案

**1.1 数据库模型支持**

路线表已有 `scrape_interval` 字段（v1.0.0已实现），默认值为6小时：

```python
# src/flightscanner/models/database.py
class Route(Base):
    # ...
    scrape_interval = Column(Integer, default=6, nullable=False)  # 小时
```

**1.2 前端UI增强**

在 `ui/components/overview.py` 中新增：

- **采集间隔显示**：在路线详情中显示当前采集间隔
- **间隔设置按钮**：使用 `st.popover()` 提供间隔调整界面
  - 支持 1/2/3/4/6/8/12/24 小时选项
  - 实时更新配置
- **立即采集按钮**：新增 🔄 按钮触发即时价格采集（调试模式）


**1.3 服务层支持**

在 `src/flightscanner/core/services/route_service.py` 中新增：

```python
def update_route_interval(self, route_id: int, new_interval: int) -> Optional[Route]:
    """更新路线的采集间隔"""
    route = self.session.query(Route).filter(Route.id == route_id).first()
    if route:
        route.scrape_interval = new_interval
        self.session.commit()
        return route
    return None
```

**1.4 调度器架构重构**

在 `src/flightscanner/scheduler/price_monitor.py` 中实现：

**原架构（v1.0.0）**：
```python
# 全局6小时间隔，所有路线统一调度
self.scheduler.add_job(
    self.scrape_active_routes,
    trigger=IntervalTrigger(hours=6),
)
```

**新架构（v1.0.1）**：
```python
# 每条路线独立调度，使用各自的 scrape_interval
def schedule_route(self, route: Route) -> None:
    """为单条路线创建独立的调度任务"""
    job_id = f"scrape_route_{route.id}"

    async def scrape_this_route():
        await self.scrape_route(route)

    self.scheduler.add_job(
        scrape_this_route,
        trigger=IntervalTrigger(hours=route.scrape_interval),
        id=job_id,
        name=f"Scrape {route.origin} → {route.destination}",
    )

def reschedule_all_routes(self) -> None:
    """重新调度所有活跃路线"""
    # 移除旧任务
    for job in self.scheduler.get_jobs():
        if job.id.startswith("scrape_route_"):
            self.scheduler.remove_job(job.id)

    # 为每条路线创建新任务
    active_routes = route_service.get_active_routes()
    for route in active_routes:
        self.schedule_route(route)
```

### 2. 智能登录流程优化

#### 问题背景

v1.0.0 版本在无头模式下遇到登录要求时的处理流程：
1. 检测到需要登录
2. 保存二维码到文件 `qunar_login_qr.png`
3. 抛出 `LoginRequiredError` 异常
4. 用户需要手动打开文件扫码
5. 扫码后需要重启程序
6. **链路太长，用户体验差**

#### 解决方案

**2.1 自动弹出二维码**

在 `src/flightscanner/scrapers/qunar_scraper.py` 中优化登录流程：

```python
async def search_flights(self, params: SearchParams) -> List[FlightPrice]:
    # ... 前置代码 ...

    if await self._is_login_required(page):
        logger.warning("Login required detected")

        # 捕获二维码
        qr_code_data = await self._capture_login_qr_code(page)
        if qr_code_data:
            qr_path = "qunar_login_qr.png"
            with open(qr_path, "wb") as f:
                f.write(base64.b64decode(qr_code_data))

            if self.headless:
                # 【新增】自动使用系统默认图片查看器打开二维码
                logger.info("Opening QR code with system viewer...")
                import subprocess, platform

                system = platform.system()
                if system == "Darwin":      # macOS
                    subprocess.Popen(["open", qr_path])
                elif system == "Windows":   # Windows
                    subprocess.Popen(["start", qr_path], shell=True)
                elif system == "Linux":     # Linux
                    subprocess.Popen(["xdg-open", qr_path])

                # 【新增】等待登录完成
                login_success = await self._wait_for_login(page, timeout=120)

                if not login_success:
                    raise LoginRequiredError("Login timeout")

                # 【新增】登录成功后自动继续执行
                logger.info("Login successful! Continuing with scraping...")
                await asyncio.sleep(2)  # 等待页面就绪

                # 【新增】保存 cookies 供下次使用
                cookies = await self._context.cookies()
                logger.info(f"Saved {len(cookies)} cookies from successful login")

    # 继续执行采集流程
    # ...
```

**2.2 登录状态轮询检测**

新增 `_wait_for_login()` 方法实现智能等待：

```python
async def _wait_for_login(self, page: Page, timeout: int = 120) -> bool:
    """等待登录完成（通过轮询检测）

    Args:
        page: Playwright页面对象
        timeout: 超时时间（秒），默认120秒

    Returns:
        bool: 登录成功返回True，超时返回False
    """
    start_time = asyncio.get_event_loop().time()
    poll_interval = 2  # 每2秒检查一次

    while (asyncio.get_event_loop().time() - start_time) < timeout:
        try:
            # 检测1：登录弹窗是否消失
            if not await self._is_login_required(page):
                logger.info("Login modal disappeared - login successful")
                return True

            # 检测2：用户信息元素是否出现
            user_info = await page.query_selector(
                "[class*='user'], [class*='username'], [class*='avatar']"
            )
            if user_info and await user_info.is_visible():
                logger.info("User info detected - login successful")
                return True

            # 等待下次检测
            await asyncio.sleep(poll_interval)

        except Exception as e:
            logger.warning(f"Error during login polling: {e}")
            await asyncio.sleep(poll_interval)

    logger.warning(f"Login timeout after {timeout} seconds")
    return False
```

## 技术实现细节

### 1. 调度器架构变更

**数据结构**：
```python
# 旧架构：单一全局任务
Job(
    id="scrape_active_routes",
    trigger=IntervalTrigger(hours=6)
)

# 新架构：多任务并行
Job(id="scrape_route_1", trigger=IntervalTrigger(hours=1))   # 紧急路线
Job(id="scrape_route_2", trigger=IntervalTrigger(hours=6))   # 普通路线
Job(id="scrape_route_3", trigger=IntervalTrigger(hours=24))  # 非紧急路线
```

**优势**：
- ✅ 灵活性：每条路线独立配置
- ✅ 性能：避免不必要的高频采集

- ✅ 可扩展性：易于添加新的调度策略

### 2. 前端状态管理

**优势**：
